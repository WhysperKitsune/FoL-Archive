Known Issue in v1.0 & v1.2:
If you have a drawing paused, and attempt to resume, some PC‚Äôs with lower specs may have trouble resuming properly. What this means is, when a drawing is resumed, the mouse moves back to its last position in the drawing, then a sleep function is called for 10 milliseconds, and then it sets the primary button of the mouse down.

This can cause ‚Äúrouge lines‚Äù to be drawn by lower spec pc‚Äôs because the mouse move command doesn‚Äôt get processed fast enough, causing the mouse down to occur before the mouse reaches its logged position.

There are multiple ways to fix this issue, and i will have an update out this weekend changing the sleep to a higher millisecond value, likely around 1000 ms (1 second). I will also allow users to modify this amount of delay in ms so higher end pc users can keep the sleep value lower to allow for that extra second of draw time, since every second counts.

(This will not be the only change coming in the next update, i have a few more additions coming to the tool in the future patch as well)

Happy Drawing
